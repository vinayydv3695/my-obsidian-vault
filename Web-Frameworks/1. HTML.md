

## 1. Introduction to HTML

- **HTML** stands for **HyperText Markup Language**
    
- It is used to **describe the structure of web pages**
    
- HTML documents are made up of **elements**
    
- Elements are represented using **HTML tags**
    
- HTML is **not a programming language**, it is a **markup language**
    

---

## 2. HTML Tags

- Tags are used to **mark up elements**
    
- Tags are enclosed within `<` and `>`
    
- Most tags come in **pairs**
    
    - Opening tag: `<b>`
        
    - Closing tag: `</b>`
        
- HTML tags are **not case-sensitive**
    
- Some tags are **self-closing**
    
    - Examples: `<img>`, `<input>`
        
- Tags can contain **attributes**
    
    - Attributes give **additional information**
        
- **Nested tags** are allowed
    
    - Rule: **Last opened tag is closed first**
        

---

## 3. Basic HTML Document Structure

### Basic Skeleton

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>
</body>
</html>
```

### Explanation

- `<!DOCTYPE html>`
    
    - Informs browser about **HTML version**
        
- `<html lang="en">`
    
    - Specifies **language of the document**
        
- `<head>`
    
    - Contains **meta information**
        
- `<meta charset="UTF-8">`
    
    - Defines **Unicode character encoding**
        
- `<title>`
    
    - Appears on **browser title bar**
        
- `<body>`
    
    - Contains **visible webpage content**
        
- HTML comments:
    

```html
<!-- This is a comment -->
```

---

## 4. Meta Tags (Important)

- **Viewport Meta Tag**
    

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

- Ensures **responsive design**
    
- Width matches **device width**
    
- Initial scale controls **zoom level**
    
- **Compatibility Meta Tag**
    

```html
<meta http-equiv="X-UA-Compatible" content="ie=edge">
```

- Supports **Microsoft Edge**
    

---

## 5. Common HTML Tags

### Text Formatting Tags

- Headings: `<h1>` to `<h6>`
    
- Paragraph: `<p>`
    
- Line break: `<br>`
    
- Bold: `<b>`
    
- Italic: `<i>`
    
- Underline: `<u>`
    

---

## 6. Anchor Tag

- Used to create **links**
    

```html
<a href="url">Text</a>
```

### Attributes

- `href` ‚Üí destination URL
    
- `target` ‚Üí where link opens
    
- `title` ‚Üí extra information (tooltip)
    

---

### Types of Lists

1. **Unordered List**
    

```html

</ul>
```

2. **Ordered List**
    


## 8. Image Tag

```html

```

- `src` ‚Üí image URL
    
- `alt` ‚Üí alternate text if image fails to load

## 9. Table Tag

```html

  </tr>

    <td>Data</td>
  </tr>
```

- `<table>` ‚Üí table container
    
- `<tr>` ‚Üí table row
    
- `<th>` ‚Üí table header
    
- `<td>` ‚Üí table cell
    

‚ö† **Table hierarchy must be maintained**

---

## 10. Forms in HTML

```html

  <input type="text">
  <input type="radio">
```

- `action` ‚Üí operation after submission
    
- `method` ‚Üí GET or POST
    
- `target` ‚Üí where result opens
    

### Input Types

- `text` ‚Üí single line input
    
- `radio` ‚Üí radio button
    
- `submit` ‚Üí submit form

## 1. Introduction to HTML5

- **HTML5** is the latest version of HTML (HTML5.2 by W3C)
    
- It **redefines semantics** of existing elements
    
- Introduces **new semantic elements**
    
- Adds native support for:
    
    - Multimedia (Audio / Video)
        
    - Form validation
        
    - Web applications
        
- Defines a **parsing algorithm** to build DOM even from poor markup
    
- HTML5 is **preferred over Flash** due to:
    
    - No plugins required
        
    - Better performance
        
    - Mobile compatibility

## 2. Applications of HTML5

- Semantic Tags
    
- Multimedia Support
    
- Form Validations
    
- Local Storage
    
- Offline Pages
    
- WebSQL
    
- Cache Storage
    
- Cookies
    
- Drag and Drop
    
- Canvas & SVG
    
- Geolocation

## 3. Semantic Elements (Concept)

- Semantic elements **describe the meaning of content**
    
- Improves:
    
    - Accessibility
        
    - SEO
        
    - Code readability
        
- Tags are enclosed in `< >`
    
- Elements may have **attributes**
    
- Attributes control **behavior of elements**
    

Example:

```html
<input type="text" name="first_name" disabled>
```

---

## 4. HTML5 Basic Document Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>Document</title>
</head>
<body>
  <h1>Hello World!</h1>
</body>
</html>
```

### Purpose

- `<!DOCTYPE html>` ‚Üí Identifies HTML5
    
- `charset="UTF-8"` ‚Üí Unicode encoding
    
- `viewport` ‚Üí Responsive design
    
- `X-UA-Compatible` ‚Üí IE rendering
    
- `<body>` ‚Üí Visible content
    

---

## 5. HTML5 Semantic Page Structure

### Main Sections

- `<header>` ‚Üí Introductory content, logo, title
    
- `<nav>` ‚Üí Navigation links
    
- `<main>` ‚Üí Dominant page content (only one allowed)
    
- `<section>` ‚Üí Thematic grouping of content
    
- `<article>` ‚Üí Independent, reusable content
    
- `<aside>` ‚Üí Indirectly related content (sidebar)
    
- `<figure>` ‚Üí Self-contained content
    
- `<figcaption>` ‚Üí Caption for figure
    
- `<footer>` ‚Üí Author, copyright, links
    

---

## 6. Semantic Elements ‚Äì Detailed

### `<header>`

- Appears at top of page/section
    
- Contains logo, title, menu, search bar
    

### `<main>`

- Represents **core content**
    
- Must be **unique**
    
- Not part of document outline
    

### `<section>`

- Logical grouping
    
- Usually has heading
    

### `<article>`

- Standalone content
    
- Can be reused or distributed independently
    

### `<nav>`

- Navigation links
    
- Used for menus, breadcrumbs
    

### `<figure>`

- Self-contained media or code
    
- Caption via `<figcaption>`
    

### `<aside>`

- Side content
    
- Not part of main flow
    

### `<footer>`

- Footer for nearest section/article
    
- Contains metadata

## 7. HTML5 Form Validations

### `required` Attribute

- Boolean attribute
    
- Prevents submission if empty
    

```html
<input type="text" required>
```

---

### `autocomplete` Attribute

- Suggests previously entered values
    
- Values: `on` | `off`
    
- Works with:
    
    - text, search, url, email, range, color
        

```html
<input autocomplete="on">
```
