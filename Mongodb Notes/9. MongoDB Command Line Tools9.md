# ## 1. Overview of MongoDB Database Tools

MongoDB provides several CLI tools for **backup, restore, import, export, and monitoring**.

### **Included Tools**

- **mongodump** → Backup (BSON format)
    
- **mongorestore** → Restore BSON backup
    
- **mongoimport** → Import JSON/CSV/TSV
    
- **mongoexport** → Export data to JSON/CSV
    
- **bsondump** → Convert BSON → JSON
    
- **mongotop** → Monitor read/write activity
    

**Download link:**  
[https://www.mongodb.com/try/download/database-tools](https://www.mongodb.com/try/download/database-tools)

---


# # **mongodump — Backup Tool**

### **Purpose**

- Creates **binary BSON backup** of databases or collections.
    
- Works with standalone, replica sets, sharded clusters, and flex clusters.
    

---

## ### Basic Backup Command

```
mongodump --db=<database> --out=<backupDirectory>
```

Example:

```
mongodump --db="OnlineShop" --collection="Users" --out=C:\dump
```

---

## ### Backup into a Single Archive

```
mongodump --archive=test.archive --db=test
```

Creates 1 single file instead of multiple BSON files.

---

## ### Backup with Authentication

```
mongodump --host <hostname> --port <port> \
--username <username> --password <password> \
--authenticationDatabase admin \
--db <database> --out <backupDirectory>
```

Using a URI:

```
mongodump --uri="mongodb://username:pass@localhost:27017/mydb?authSource=admin" \
--archive=backup.archive
```

# # **mongorestore — Restore Tool**

### **Purpose**

Restores data from:

- Dump folder
    
- Archive file
    

Recreates collections and indexes.

---

## ### Basic Restore

```
mongorestore ./dump/
```

### **Restore from Archive**

```
mongorestore --archive=backup.archive
```

---

## ### Restore DB With New Name

```
mongodump --archive="dump-test-db" --db=test
mongorestore --archive="dump-test-db" \
--nsFrom="test.*" --nsTo="examples.*"
```

---

## ### Restore With Authentication

```
mongorestore --username <user> --password <pass> \
--authenticationDatabase admin \
--db <database> ./dump/<database>
```

---

## ### Restore Specific Collection

```
mongorestore --nsInclude=test.purchaseorders ./dump
```

# # **bsondump — BSON → JSON**

### **Purpose**

Converts BSON files into **readable JSON**.

Example:

```
bsondump collection.bson > collection.json
```

Useful before doing restores.

---

# --------------------------------------------------------

# # **mongoimport — Import Tool**

### **Purpose**

Imports data from:

- JSON
    
- CSV
    
- TSV
    

### **Basic Command**

```
mongoimport --db=users --collection=contacts --file=contacts.json
```

### **Import Modes**

- **upsert** → Insert or update
    
- **merge** → Merge fields
    
- **delete** → Delete matching documents
    

---

# --------------------------------------------------------

# # **mongoexport — Export Tool**

Exports MongoDB collections to:

- JSON
    
- CSV
    

### **Example**

```
mongoexport --collection=events --db=reporting --out=events.json
```

---

# --------------------------------------------------------

# # **mongotop — Monitoring Tool**

Tracks time spent **reading & writing** data by each collection.

### **Usage**

```
mongotop
```

Displays:

- Namespace
    
- Read time
    
- Write time
    
- Total time
    

Refreshes every second.

---

# # **Backup & Restore Workflow**

### **1. Backup**

```
mongodump ...
```

### **2. Transfer Archive**

Move dump or archive to destination.

### **3. Restore**

```
mongorestore ...
```

---

# --------------------------------------------------------

# # **Import & Export Workflow**

### **Export**

```
mongoexport ...
```

### **Import**

```
mongoimport ...
```

---

# --------------------------------------------------------

# # **Best Practices**

- Always authenticate before backup/restore
    
- Use archives for portability
    
- Automate backups in scripts
    
- Test restores before production failures
    
- Use **mongotop** to monitor performance regularly
