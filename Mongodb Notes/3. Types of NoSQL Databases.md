# ## 1. Key-Value Databases

A Key-Value database stores data as **key–value pairs**, similar to a **dictionary** or **map**.

### **Structure Example**

```
{
  "user:100": {
    "name": "john",
    "age": 20
  },
  "items:3987": ["item1", "item2", "item3", "item4"]
}
```

### **Characteristics**

- Extremely fast read/write operations
    
- Direct lookups using key
    
- Highly scalable (vertical + horizontal)
    
- Simple data model
    
- Not suitable for complex querying
    

### **Use Cases**

- Caching
    
- Session management
    
- Shopping cart data
    
- Real-time applications
    

### **Examples**

- **Redis**
    
- **DynamoDB**
    

---

# ## 2. Column Family Databases (Wide Column Stores)

Data is stored in **columns**, not rows.  
Tables → Column Families → Columns.

### **Advantages of Column-Oriented Systems**

- Highly efficient for analytical queries
    
- Excellent for large-scale, distributed systems
    
- Writes are optimized heavily
    
- Can store sparse data efficiently
    

### **Use Cases**

- Time-series data (temperature readings)
    
- Recommendation engines (page views)
    
- User activity logs
    
- IoT applications
    

### **Examples**

- **Apache Cassandra** (massive write throughput)
    
- **Apache HBase**
    
- **ScyllaDB**
    

---

# ## 3. Document Databases

Store data as **documents** (JSON, BSON, XML, YAML).

### **Structure Example**

```
{
  "Employee ID": "123",
  "First Name": "Abhishek",
  "Last Name": "Sharma"
}
```

### **Characteristics**

- No rigid schema
    
- Highly flexible
    
- Supports nested data
    
- Powerful query capabilities
    
- Great for real-time apps and unstructured data
    

### **Use Cases**

- APIs and microservices
    
- CRM systems
    
- User profiles
    
- Product catalogs
    
- Content management
    

### **Examples**

- **MongoDB**
    
- **CouchDB**
    

---

# ## 4. Graph Databases

Stores data as **nodes** and **edges** (relationships).

### **Characteristics**

- Designed for highly connected data
    
- Efficient traversal of relationships
    
- Ideal for network-like structures
    

### **Use Cases**

- Social networks
    
- Fraud detection
    
- Recommendation systems
    
- Knowledge graphs
    

### **Examples**

- **Neo4j** (uses Cypher Query Language)
    
- **Amazon Neptune**
    
- **ArangoDB**
